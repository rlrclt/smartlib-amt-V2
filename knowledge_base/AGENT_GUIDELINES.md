2. **โฟกัสพื้นที่งานเท่านั้น**: ทำงานเฉพาะในโปรเจกต์ `smartlib-amt-V2` เท่านั้น (Path: `/home/yoru/projects/smartlib-amt-V2`)
3. **ใช้ rtk นำหน้าเสมอ**: ทุกคำสั่ง Shell ต้องขึ้นต้นด้วย `rtk` เพื่อบีบอัดข้อมูลและประหยัด Token

## 🛠️ การใช้งาน rtk (Rust Token Killer)
โปรเจกต์นี้ติดตั้ง `rtk` เวอร์ชัน 0.37.2 ไว้แล้ว ให้ใช้ดังนี้:
- **สำรวจโครงสร้าง**: `rtk tree` หรือ `rtk ls`
- **อ่านไฟล์สะอาด**: `rtk read <path_to_file>`
- **เช็ค Git**: `rtk git status` หรือ `rtk git diff`
- **เช็คความคุ้มค่า**: `rtk gain` (ดูจำนวน Token ที่ประหยัดได้)
- **สรุป Error**: หากคำสั่งพัง ให้รันซ้ำโดยใช้ `rtk` ครอบเพื่อให้มันสรุปจุดที่ผิด

## ⚡ กลยุทธ์ประหยัด Token (Token-Optimizer)
ใช้หลักการ **"อ่านน้อยที่สุด -> เข้าใจมากที่สุด"**
- **ห้ามอ่านทั้งไฟล์**: หากไฟล์เกิน 100 บรรทัด ห้ามใช้ `read_file` ทั้งหมด
- **วิธีอ่านที่ถูกต้อง**:
    1. ใช้ `rtk grep -n "keyword" file` เพื่อหาเลขบรรทัด
    2. ใช้ `sed -n 'start,endp' file` เพื่ออ่านเฉพาะจุดที่เกี่ยวข้อง
- **ตรวจสอบก่อนแก้ไข**: ใช้คำสั่งเช็ค Syntax เช่น `node --check file.js` ก่อนบันทึกไฟล์เสมอ

## 🚀 กระบวนการทำงานระดับมืออาชีพ (Professional Workflow)
AI Agent ต้องทำงานตาม Lifecycle นี้เสมอ:
1. **Research (วิจัย)**: ค้นหาและทำความเข้าใจ Codebase อย่างละเอียดก่อนเสนอการแก้ไข (ใช้ `rtk grep`, `rtk find`) ห้ามเดาหรือสมมติโครงสร้างโค้ดเอง
2. **Strategy (วางแผน)**: สรุปแผนการทำงาน (Step-by-step plan) ให้ User ตรวจสอบก่อนเริ่ม Execute ไฟล์ใดๆ
3. **Execution (ดำเนินการ)**: แก้ไขโค้ดแบบ "Surgical Edits" (เน้น `replace` เฉพาะจุดที่จำเป็น) ห้าม Rewrite ไฟล์ใหม่ทั้งหมดหากไม่จำเป็นจริงๆ
4. **Validation (ตรวจสอบ)**: **สำคัญที่สุด** งานจะเสร็จสมบูรณ์เมื่อผ่านการตรวจสอบดังนี้เท่านั้น:
    - รันคำสั่งเช็ค Syntax (เช่น `node --check`, `npx tsc --noEmit`)
    - รัน Automated Tests ที่เกี่ยวข้อง (เช่น `rtk npm test -- <file>`)
    - ตรวจสอบความถูกต้องด้วยตนเองผ่าน `rtk git diff`
