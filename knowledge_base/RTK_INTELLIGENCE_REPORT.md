# RTK (Runtime Toolkit) Intelligence Report

This document outlines the capabilities, usage patterns, and efficiency benefits of the **RTK (Runtime Toolkit)** as observed and practiced by the Gemini CLI agent.

## 1. Overview
RTK is an optimization layer for shell commands designed to enhance the interaction between AI agents and complex development environments. Its primary goal is to maximize **context efficiency**—ensuring the agent can see more of the codebase while using fewer tokens.

## 2. Core Command Intelligence

### 🛠️ Execution Commands
- **`rtk <command>`**: The universal prefix. It intercepts standard commands to provide compressed, AI-optimized output.
- **`rtk read <file>`**: A surgical alternative to `cat` or `read_file`. It strips non-essential metadata and provides a clean view of the source code.
- **`rtk tree`**: Generates a compressed directory map. 
    - *Usage*: `rtk tree --ultra-compact` is essential for large projects to prevent context overflow.

### 📊 Efficiency & Analytics
- **`rtk gain`**: Provides real-time statistics on token savings.
    - *Metric*: In this project, we achieved a **~32.3% reduction** in token usage across nearly 1,000 commands.
    - *Impact*: Higher impact commands (like `git diff`) often see savings of **60-80%**.
- **`rtk deps`**: Quickly summarizes project dependencies (parsing `package.json`, `requirements.txt`, etc.) without reading the entire file.

## 3. Best Practices for AI Agents

1.  **Mandatory Prefixing**: Every `run_shell_command` should start with `rtk` to ensure the session remains lean.
2.  **Compact Navigation**: Use `rtk tree` before `glob` or `list_directory` when exploring unknown structures.
3.  **Diff Monitoring**: Use `rtk git diff` to review changes; it provides a highly focused summary that is easier for the AI to validate.

## 4. Why RTK is Critical for smartlib-amt-V2
Given the hybrid nature of this project (Static SPA + Google Apps Script), the codebase spans multiple environments. RTK allows the agent to:
- Monitor both `public/` and `apps_script/` simultaneously without hitting token limits.
- Quickly cross-reference documentation in `docs/` while analyzing code in `public/app/views/`.
- Maintain a clear "mental map" of the project's evolving architecture.
