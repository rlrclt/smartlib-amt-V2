## 2. การปรับปรุงกระบวนการ (Flow Redesign)

### 2.1 Mandatory Check-in Flow
- **Interceptor (Guard)**: เพิ่ม Logic ตรวจสอบสถานะการเข้าใช้ (`hasActiveVisit`) ใน `router.js` และ `member_shell.js`
- **Navigation Guard**: หากสมาชิกไม่มี `activeVisit` ระบบจะ Redirect ไปที่ `/app/checkin` โดยอัตโนมัติ (ยกเว้นหน้าเช็คอิน)
- **Check-in UI**: ปรับหน้าเช็คอินให้เป็นจุดเริ่มต้น (Gateway) ของระบบ

### 2.2 Check-out Flow
- **Sticky Status Bar**: เพิ่มแถบสถานะ (Active Visit Bar) ที่ด้านบนของ `member_shell.js` (หากมี visit ค้างอยู่)
- **Manual Check-out**: เพิ่มปุ่ม "เช็คเอาท์" ในแถบสถานะ เพื่อให้สมาชิกจบ session ได้จากทุกหน้า โดยไม่ต้องกลับไปที่หน้าเช็คอิน
- **Auto Check-out**: คงระบบ Auto-close ของ Backend ไว้เป็น Fail-safe

### 4.1 Route Guard (Router Middleware)
```javascript
// ใน router.js หรือฟังก์ชันที่ใช้จัดการ route
async function guard(route) {
  const visit = await apiCheckActiveVisit(); // ดึงสถานะจาก Backend
  if (!visit.hasActive && route !== '/app/checkin') {
    return '/app/checkin'; // บังคับไปหน้าเช็คอิน
  }
  return route;
}
```
