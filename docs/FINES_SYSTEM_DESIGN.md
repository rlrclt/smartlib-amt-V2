# แผนการออกแบบ: ระบบจัดการค่าปรับ (Fines Management System)

เอกสารนี้ระบุรายละเอียดการพัฒนาโมดูลจัดการค่าปรับที่เกิดขึ้นจากการยืม-คืนหนังสือ เช่น การคืนหนังสือล่าช้า หรือหนังสือชำรุด/สูญหาย

## 1. วัตถุประสงค์
เพื่อเป็นระบบบันทึก ตรวจสอบ และชำระเงินค่าปรับที่โปร่งใส โดยเชื่อมโยงกับประวัติการยืม-คืน

## 2. โครงสร้างข้อมูล (Database Schema)
ตารางหลัก: **`fines`** ใน Google Sheets

| Column | Field Name | Type | Description |
| :--- | :--- | :--- | :--- |
| **A** | `fineId` | String | **Primary Key**: รหัสค่าปรับ (เช่น `FN-20240525-0001`) |
| **B** | `loanId` | String | **Foreign Key**: เชื่อมกับ `loans.loanId` |
| **C** | `uid` | String | **Foreign Key**: เชื่อมกับ `users.uid` (ผู้ที่ต้องชำระ) |
| **D** | `amount` | Number | จำนวนเงินค่าปรับ |
| **E** | `type` | Enum | ประเภท: `overdue`, `damaged`, `lost` |
| **F** | `status` | Enum | สถานะ: `unpaid` (ยังไม่จ่าย), `paid` (จ่ายแล้ว), `waived` (ยกเว้นให้) |
| **G** | `paidAt` | ISO8601 | วันที่และเวลาที่ชำระเงิน |
| **H** | `receivedBy` | String | รหัสผู้ใช้ที่รับชำระเงิน (Admin/Librarian) |
| **I** | `notes` | String | หมายเหตุเพิ่มเติม |
| **J** | `createdAt` | ISO8601 | วันที่สร้างรายการค่าปรับ |
| **K** | `updatedAt` | ISO8601 | วันที่อัปเดตข้อมูลล่าสุด |
| **L** | `barcode` | String | บาร์โค้ดหนังสือ (Denormalized เพื่อการแสดงผลเร็ว) |
| **M** | `bookTitle` | String | ชื่อหนังสือ (Denormalized เพื่อการแสดงผลเร็ว) |

⚠️ **หมายเหตุสำคัญ**: คอลัมน์ `loans.fineAmount` (Col N) ในตารางยืม-คืน ใช้สำหรับเก็บยอดค่าปรับสะสมแบบ Real-time เพื่อแสดงผลในหน้าเว็บอย่างรวดเร็ว ส่วนตาราง `fines` นี้ใช้สำหรับเป็น "หลักฐานใบเสร็จ" และการทำธุรกรรมทางการเงินที่แน่นอน ห้ามนำยอดจากสองที่มานับรวมกัน

## 3. กฎและการคำนวณ (Fine Logic)

### 3.1 การสร้างรายการค่าปรับอัตโนมัติ (Overdue Fine)
- เมื่อมีการคืนหนังสือล่าช้า (`returnDate` > `dueDate`) ระบบจะคำนวณค่าปรับ:
  - `จำนวนวันที่เกิน` x `อัตราค่าปรับต่อวัน (จาก Settings)`
  - สร้างแถวใหม่ในตาราง `fines` พร้อมสถานะ `unpaid`

### 3.2 การจัดการโดยเจ้าหน้าที่ (Manual Fine)
- กรณีหนังสือชำรุด (`damaged`) หรือสูญหาย (`lost`):
  - เจ้าหน้าที่สามารถระบุจำนวนเงินค่าปรับตามดุลยพินิจหรือตามราคาหนังสือ
  - สร้างรายการในตาราง `fines` และระบุหมายเหตุ

### 3.3 การชำระเงิน (Payment Process)
- สมาชิกต้องชำระเงินที่เคาน์เตอร์ห้องสมุด
- เจ้าหน้าที่ทำการบันทึกการรับเงิน อัปเดตสถานะเป็น `paid` และบันทึกวันที่/ผู้รับเงิน

### 3.4 การยกเว้นค่าปรับ (Waive Process)
- **สิทธิ์การดำเนินการ**: เฉพาะผู้ใช้ที่มีบทบาท `admin` เท่านั้น (บรรณารักษ์ทั่วไปไม่มีสิทธิ์ยกเว้นค่าปรับ)
- **ข้อบังคับ**: ต้องระบุเหตุผลในช่อง `notes` ทุกครั้ง (ห้ามว่าง)
- **การบันทึก**: ระบบจะบันทึกรหัส `uid` ของ Admin ผู้ทำการยกเว้นในฟิลด์ `receivedBy`

## 4. ความสัมพันธ์ของข้อมูล (Data Relationships)
1. **`fines.loanId` ↔ `loans.loanId`**: เพื่ออ้างอิงว่าค่าปรับนี้มาจากประวัติการยืมครั้งไหน
2. **`fines.uid` ↔ `users.uid`**: เพื่อแสดงรายการค่าปรับค้างชำระในหน้าโปรไฟล์ของสมาชิก

## 5. การตั้งค่าที่เกี่ยวข้อง (Settings Integration)
Admin สามารถกำหนดค่าเหล่านี้ได้ที่หน้า `/manage/settings`:
- `fine_rate_overdue`: อัตราค่าปรับคืนสายต่อวัน (บาท)
- `fine_buffer_days`: จำนวนวันที่อนุญาตให้เกินกำหนดโดยไม่คิดค่าปรับ (ถ้ามี)
