# แผนการออกแบบ: บริการสมาชิกและระบบรายงานห้องสมุด (MVP Spec)

อัปเดตล่าสุด: 2026-04-30  
สถานะ: Draft for Implementation (Actionable MVP)

เอกสารนี้สรุปความต้องการจาก `docs/fig-bug/gg.md` และจัดโครงสร้างใหม่ให้พร้อมใช้งานสำหรับทีมพัฒนาและทีมบริหารห้องสมุด

## 1. วัตถุประสงค์
- เพิ่มขั้นตอนเลือก "บริการที่ต้องการใช้" หลังสมาชิกเข้าสู่ระบบ
- รองรับการเพิ่มลิงก์ E-book ในหน้าจัดการหนังสือสำหรับเจ้าหน้าที่
- รองรับชุดรายงานหลักเพื่อการบริหารงานรายวัน รายเดือน รายปี และรายกลุ่ม

## 2. User Flow (MVP)

### 2.1 Member: เลือกบริการหลัง Login
1. สมาชิก Sign in สำเร็จ
2. ระบบแสดง Popup: "กรุณาเลือกบริการที่คุณต้องการ"
3. สมาชิกเลือกได้หลายบริการ (multi-select)
4. กดยืนยันแล้วจึงเข้าหน้าหลักของสมาชิกได้

บริการขั้นต่ำที่ต้องมีใน MVP:
- ยืม-คืนหนังสือ
- จองหนังสือ และ E-book
- ใช้งาน E-book 

กฎธุรกิจขั้นต่ำ:
- ต้องมีอย่างน้อย 1 บริการก่อนกดยืนยัน
- เก็บค่าที่เลือกไว้เพื่อใช้ในการวิเคราะห์รายงาน

### 2.2 Staff/Admin: เพิ่มหนังสือใหม่พร้อมลิงก์ E-book
1. เจ้าหน้าที่ไปหน้าเพิ่มหนังสือใหม่
2. ระบบรองรับ field สำหรับลิงก์ E-book
3. หากมีการกรอกลิงก์ ให้ตรวจสอบรูปแบบ URL ขั้นต่ำ
4. บันทึกข้อมูลพร้อม metadata หนังสือเดิม

## 3. Reporting Catalog (MVP)

### R1: รายงานจำนวนสมาชิกที่มาใช้ห้องสมุด
- วัตถุประสงค์: ดูแนวโน้มผู้เข้าใช้
- Filter: รายวัน / รายเดือน / รายปี
- Output ขั้นต่ำ: วันที่/ช่วงเวลา, จำนวนสมาชิกเข้าใช้

### R2: รายงานจำนวนสมาชิกตามประเภทบริการที่ใช้
- วัตถุประสงค์: วิเคราะห์ความนิยมบริการ
- Filter: รายวัน / รายเดือน / รายปี
- Output ขั้นต่ำ: ประเภทบริการ, จำนวนสมาชิกที่เลือกใช้

### R3: รายงานสมาชิกสมัครใหม่
- วัตถุประสงค์: ติดตามการเติบโตของสมาชิก
- Filter: รายวัน / รายเดือน / รายปี
- Output ขั้นต่ำ: จำนวนสมัครใหม่, รายชื่อสมาชิกที่สมัครในช่วงเวลา

### R4: รายงานการยืมหนังสือของนักเรียน
- วัตถุประสงค์: วิเคราะห์การยืมรายบุคคลและรายสาขา
- Filter: รายบุคคล, รายสาขาวิชา, ช่วงเวลา
- Output ขั้นต่ำ: รหัสสมาชิก/ชื่อ, สาขาวิชา, จำนวนครั้งยืม, รายการหนังสือ

### R5: รายงานหนังสือใหม่/เก่า และสถานะคัดออก
- วัตถุประสงค์: วางแผนจัดการคลังหนังสือ
- นิยาม:
  - หนังสือใหม่: ปีพิมพ์หรือปีนำเข้า = ปีปัจจุบัน
  - หนังสือเก่า: ปีก่อนหน้าปีปัจจุบัน
  - หนังสือควรพิจารณาจำหน่าย: อายุหนังสือ >= 3 ปี
- Filter: รายวัน / รายเดือน / รายปี
- Output ขั้นต่ำ: จำนวนหนังสือใหม่, จำนวนหนังสือเก่า, รายการหนังสือที่เข้าเกณฑ์พิจารณาจำหน่าย

### R6: รายงานรายชื่อเจ้าหน้าที่
- วัตถุประสงค์: ตรวจสอบผู้ปฏิบัติงานตามบทบาท
- Output ขั้นต่ำ: รายชื่อเจ้าหน้าที่, สถานะใช้งาน

### R7: รายงานรายชื่อแอดมิน
- วัตถุประสงค์: ตรวจสอบสิทธิ์ระดับสูง
- Output ขั้นต่ำ: รายชื่อแอดมิน, สถานะใช้งาน

### R8: รายงานสมาชิกรายบุคคล
- วัตถุประสงค์: ดูภาพรวมประวัติการใช้งานของสมาชิก
- Filter: สมาชิกรายคน
- Output ขั้นต่ำ: โปรไฟล์สมาชิก, ประวัติการยืม, ประวัติการเข้าใช้บริการ

## 4. Acceptance Criteria (MVP)
- หลัง Login ต้องมี popup เลือกบริการ และเลือกหลายบริการได้จริง
- ห้ามผ่าน popup หากยังไม่เลือกบริการใดเลย
- หน้าจอเพิ่มหนังสือรองรับ field E-book URL และบันทึกได้
- สร้างรายงาน R1-R8 ได้ครบ พร้อม filter ขั้นต่ำตามที่ระบุ
- รายงานทุกตัวมีข้อมูลรวมและข้อมูลรายละเอียดที่ตรวจสอบย้อนหลังได้

## 5. ข้อกำหนดข้อมูล/อินเทอร์เฟซขั้นต่ำ
- พารามิเตอร์รายงานมาตรฐาน:
  - `period`: `daily | monthly | yearly`
  - `from`, `to`: ช่วงวันที่
  - `memberId`: สำหรับรายงานรายบุคคล
  - `department`: สำหรับรายงานรายสาขาวิชา
  - `role`: สำหรับรายงานรายชื่อเจ้าหน้าที่/แอดมิน
- โครงผลลัพธ์ขั้นต่ำ:
  - `summary`: ตัวเลขรวม
  - `rows`: รายการรายละเอียด

## 6. Out of Scope (รอบนี้ยังไม่รวม)
- นิยาม UI รายงานเชิงกราฟเต็มรูปแบบ
- การส่งออกไฟล์ (PDF/Excel) ระดับ production
- กติกา retention/archival เชิงลึก
