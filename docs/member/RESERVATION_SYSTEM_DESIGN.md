# แผนการออกแบบ: ระบบการจองและนัดหมายยืมล่วงหน้า (Reservation & Pre-booking Design)

เอกสารนี้ระบุรายละเอียดการพัฒนาหน้าจอ `/app/reservations` และระบบจัดการคิวจองหนังสือ โดยเน้นการนัดหมายวันเข้ายืมล่วงหน้าและความเชื่อมโยงกับระบบพิกัด (Geofencing) และนโยบายห้องสมุด

## 1. วัตถุประสงค์
- เพื่อให้สมาชิกสามารถนัดหมายวันเข้ายืมหนังสือล่วงหน้าได้ (Pre-booking)
- เพื่อจัดการลำดับคิว (Queue) เมื่อหนังสือที่ต้องการถูกยืมออกไปหมด
- เพื่อแจ้งเตือนสมาชิกโดยอัตโนมัติเมื่อถึงคิวและหนังสือพร้อมให้บริการ

---

## 2. โครงสร้างข้อมูล (Data Architecture)

### 2.1 ตาราง `reservations` (ฐานข้อมูลหลัก)
| Field | Type | Description |
| :--- | :--- | :--- |
| `resId` | String | PK: รหัสการจอง (เช่น `RS-20260525-0001`) |
| `bookId` | String | FK: รหัสหนังสือแม่ (จาก `books_catalog`) |
| `uid` | String | FK: รหัสสมาชิก (จาก `users`) |
| `resDate` | ISO8601 | วันที่ทำรายการจอง |
| `plannedDate` | Date | วันที่สมาชิกนัดหมายจะเข้ามายืม |
| `plannedDuration`| Number | จำนวนวันที่ต้องการยืม (ต้องไม่เกิน `loanDays` ใน Policy) |
| `status` | Enum | `waiting` (รอคิว), `ready` (พร้อมรับ), `completed` (รับแล้ว), `cancelled` (ยกเลิก), `expired` (หมดเวลาจอง) |
| `holdUntil` | Date | วันสุดท้ายที่จะเก็บหนังสือไว้ให้ (คำนวณจาก `readyAt + holdDays`) |
| `notifiedAt` | ISO8601 | วันที่ส่งการแจ้งเตือนล่าสุด |

### 2.2 การปรับปรุงตาราง `settings_policies`
เพิ่มคอลัมน์เพื่อควบคุมการจอง:
- `resQuota`: จำนวนรายการจองสูงสุดที่สมาชิกถือครองได้พร้อมกัน
- `holdDays`: จำนวนวันที่ห้องสมุดจะเก็บหนังสือ "พร้อมรับ" ไว้ให้ก่อนยกเลิกคิวอัตโนมัติ

## 3. ลำดับขั้นตอนการทำงาน (Workflow Logic)

### 3.1 การสร้างรายการจอง (Booking Process)
1. **Selection**: สมาชิกเลือกหนังสือในหน้า `/app/books`
2. **Availability Check**: ระบบแสดงสถานะ "ว่าง (พร้อมจองทันที)" หรือ "ไม่ว่าง (คาดว่าจะว่างวันที่ [Date])"
3. **Input**: สมาชิกเลือก **วันที่นัดหมาย** และ **จำนวนวันยืม**
4. **Validation**:
    - ตรวจสอบโควตาการจองค้างส่ง (`resQuota`)
    - ตรวจสอบว่าสมาชิกไม่ได้ยืมเรื่องนี้อยู่แล้ว
    - ตรวจสอบความขัดแย้ง: หากเลือกวันนัดหมายทับซ้อนกับคิวอื่น ระบบจะแนะนำวันถัดไปที่ว่างเร็วที่สุด
5. **Final Action**: บันทึกข้อมูลและแจ้งลำดับคิว (ถ้ามี)

### 3.2 ระบบจัดการคิวอัตโนมัติ (Queue Management)
- **เมื่อมีการคืนหนังสือ**: Backend จะตรวจสอบคิวจองที่เก่าที่สุดที่มีสถานะ `waiting` สำหรับ `bookId` นั้น
- **การเปลี่ยนสถานะเป็น Ready**:
    - เปลี่ยนสถานะการจองเป็น `ready`
    - ตั้งค่า `holdUntil` ตามนโยบาย `holdDays`
    - ล็อคเล่มหนังสือ (`book_items`) ให้เป็นสถานะ `reserved` (เพื่อป้องกันคนอื่นมาสแกนยืมไปก่อน)
    - ส่ง Notification: "หนังสือที่ท่านจองไว้พร้อมรับแล้ว! กรุณามารับก่อนวันที่ [Date]"

### 3.3 ระบบล้างคิวหมดอายุ (Daily Expiry)
- **Trigger**: ระบบทำงานทุกวันเวลา 00:00 น.
- **Action**: รายการที่สถานะเป็น `ready` และเลยกำหนด `holdUntil` จะถูกเปลี่ยนเป็น `expired`
- **Next in Queue**: ระบบจะทำการ Match คิวถัดไปให้โดยอัตโนมัติ

## 4. รายละเอียดหน้าจอสมาชิก (Frontend: /app/reservations)

### 4.1 ส่วนแสดงผลรายการจองปัจจุบัน (Active Reservations)
- **Queue Status Display**:
    - **รอคิว (Waiting)**: แสดงลำดับที่เท่าไหร่ และวันคาดการณ์ (ETA)
    - **พร้อมรับ (Ready)**: แสดงเลขที่ตู้/เคาน์เตอร์ และวันหมดเขตการเก็บหนังสือ (Countdown)
- **Visual List**: การ์ดหนังสือที่มีรายละเอียดวันนัดหมายและระยะเวลาที่เลือกไว้
- **Action Buttons**: ปุ่ม "ยกเลิกการจอง" และปุ่ม "ดูรายละเอียดนัดหมาย"

### 4.2 ส่วนประวัติการจอง (Reservation History)
- แสดงรายการที่จบไปแล้ว (รับแล้ว/ยกเลิก/หมดอายุ) เพื่อความโปร่งใส

## 5. การรับมือกรณีพิเศษ (Edge Cases & Security)

| สถานการณ์ | การตอบสนองของระบบ |
| :--- | :--- |
| **คนยืมเดิมต่ออายุ (Renew)** | ระบบจะอัปเดตวันคาดการณ์ (ETA) ของผู้จองในคิวโดยอัตโนมัติ และส่งแจ้งเตือน "วันรับหนังสืออาจเลื่อนออกไป" |
| **หนังสือสูญหาย (Lost)** | ยกเลิกคิวจองทั้งหมดของเล่มนั้น และแจ้งสมาชิกให้เลือกเรื่องอื่นแทน |
| **จองแล้วไม่มารับบ่อยครั้ง** | (นโยบายอนาคต) อาจมีการพักสิทธิ์การจองหากมีประวัติ `expired` เกิน 3 ครั้งใน 1 เดือน |
| **จองเล่มที่ว่างอยู่** | ระบบจะทำการล็อคเล่มนั้นทันที (Status: `reserved`) และเริ่มนับ `holdUntil` ทันที |

---

## 6. ความสัมพันธ์กับระบบอื่น (System Integration)

- **SCHEMA_BOOKS.md**: ใช้สถานะ `reserved` ในตาราง `book_items` เพื่อล็อคเล่มหนังสือให้ผู้จอง
- **LOAN_SYSTEM_DESIGN.md**: 
    - อัปเดตตาราง `settings_policies` เพื่อเก็บค่า `resQuota` และ `holdDays`
    - แก้ไข `returnBook_` ให้ตรวจสอบคิวจองและเปลี่ยนสถานะเป็น `reserved` แทน `available` หากมีคิว
    - แก้ไข `loanBook_` ให้ตรวจสอบสิทธิ์กรณีเล่มหนังสือเป็น `reserved` (ต้องเป็นเจ้าของคิว)
- **CANVAS_RESERVATIONS_SPEC.md**: รายละเอียด UX/UI สำหรับการสร้างหน้าจอ `/app/reservations`
- **PROFILE_NOTIFICATION_PLAN.md**: เพิ่มประเภทการแจ้งเตือน `reservation` (Section 2.2)
