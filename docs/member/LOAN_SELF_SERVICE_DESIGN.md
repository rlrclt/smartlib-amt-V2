# แผนการออกแบบ: หน้าจอยืม-คืนด้วยตนเอง (Self-service Loan/Return Interface)

เอกสารนี้ระบุรายละเอียดการพัฒนาหน้าจอ `/app/loan-self` สำหรับสมาชิก โดยเน้นการแก้ปัญหาเรื่องพิกัด (Geofencing UX) และความรวดเร็วในการทำรายการ รวมถึงความเชื่อมโยงกับระบบ Check-in

## 1. ระบบพิกัดและสถานะการเข้าใช้ (Location & Visit Verification)

เพื่อให้การทำรายการมีความสมบูรณ์และเป็นไปตามระเบียบ สมาชิกต้องผ่านการตรวจสอบ 2 ชั้นก่อนเริ่มทำรายการ:

### 1.1 การตรวจสอบสถานะการเข้าใช้ (Visit Session Check) **[NEW]**
- **Condition**: สมาชิกต้องมี `active session` ในตาราง `library_visits` เท่านั้นถึงจะอนุญาตให้ทำรายการยืม-คืนได้
- **UX Flow**:
    - หากเข้าหน้าจอนี้แล้วไม่พบ Session ที่กำลังใช้งานอยู่ -> ระบบจะแสดง **Full-screen Overlay** พร้อมข้อความ *"กรุณาเช็คอินเข้าใช้งานห้องสมุดก่อนทำรายการ"* และปุ่ม [ไปหน้าเช็คอิน] เพื่อนำทางไปที่ `/app/checkin`
    - หากมี Session แล้ว -> อนุญาตให้ไปขั้นตอนการตรวจสอบพิกัดถัดไป

### 1.2 การแสดงผล Mini-map และพิกัด
- **Visual Interface**: แสดงแผนที่ขนาดเล็กที่มีหมุด (Marker) 2 ประเภท (User และ Authorized Points)
- **Distance Guidance**: แสดงเส้นประเชื่อมโยงไปยังจุดที่ใกล้ที่สุดพร้อมระยะห่างแบบ Real-time
- **Geofencing Verification**: ระบบจะใช้ค่า `range_borrow` และ `range_return` จากตาราง `settings_locations` ในการตรวจสอบสิทธิ์

## 5. ลำดับขั้นตอนการทำงาน (Four-Step Workflow Logic) **[UPDATED]**

### ขั้นตอนที่ 0: ตรวจสอบการเข้าใช้ (Visit Check)
- ระบบเรียก API เพื่อดูว่า `uid` นี้มีสถานะ `active` ในห้องสมุดหรือไม่
- หากไม่มี -> Redirect ไป `/app/checkin`

### ขั้นตอนที่ 1: ตรวจสอบพิกัด (Location Verification)
- เมื่อผ่านขั้นตอนที่ 0 ระบบจะทำการร้องขอพิกัดและตรวจสอบกับจุดบริการที่ใกล้ที่สุดอัตโนมัติ

### ขั้นตอนที่ 2: เลือกประเภทรายการ (Operation Selection)
- เลือก "ยืมหนังสือ" (Borrow) หรือ "คืนหนังสือ" (Return)

### ขั้นตอนที่ 3: จัดการรายละเอียดและสแกน (Action & Detail - Cart System)
- ใช้ระบบ **Scan to Cart** เพื่อความรวดเร็ว
