# แผนการออกแบบ: แดชบอร์ดผู้ดูแลระบบ (Librarian/Admin Dashboard Design)

เอกสารนี้ระบุรายละเอียดการพัฒนาหน้าจอ `/manage` ซึ่งเป็นจุดยุทธศาสตร์หลัก (Command Center) สำหรับบรรณารักษ์และผู้ดูแลระบบ เพื่อให้เห็นภาพรวมของห้องสมุดและจัดการงานที่คั่งค้างได้อย่างรวดเร็ว

## 1. วัตถุประสงค์
- เพื่อแสดงสถิติที่สำคัญของห้องสมุดแบบ Real-time
- เพื่อรวบรวมรายการที่ต้องดำเนินการ (Pending Tasks) ไว้ในที่เดียว
- เพื่อเป็นทางลัด (Quick Actions) ไปยังฟังก์ชันที่ใช้งานบ่อย

---

## 2. โครงสร้างข้อมูลสถิติ (Statistics Data)

ระบบจะดึงข้อมูลสรุปผ่าน API `manage_dashboard_stats` โดยมีตัวเลขที่สำคัญดังนี้:

| กลุ่มสถิติ | รายละเอียด | แหล่งข้อมูล |
| :--- | :--- | :--- |
| **หนังสือ (Books)** | จำนวนเล่มทั้งหมด, พร้อมใช้งาน, ถูกยืมอยู่, ชำรุด/หาย | `book_items` |
| **การยืม-คืน (Loans)** | ยืมวันนี้, คืนวันนี้, เกินกำหนดส่ง (Overdue) | `loans` |
| **สมาชิก (Members)** | สมาชิกทั้งหมด, สมาชิกใหม่สัปดาห์นี้, สมาชิกที่รอยืนยัน | `users` |
| **การเข้าใช้ (Visits)** | ผู้ใช้ในห้องสมุดขณะนี้ (Active), จำนวนเช็คอินวันนี้ | `library_visits` |
| **การเงิน (Fines)** | ยอดค่าปรับค้างชำระทั้งหมด, จำนวนรายการที่ยังไม่จ่าย | `fines` |
| **การจอง (Reservations)** | จำนวนคิวที่รออยู่, รายการที่พร้อมรับแต่ยังไม่มาเอา | `reservations` |

---

## 3. องค์ประกอบหน้าจอ (UI Components)

### 3.1 ส่วนบน: สรุปตัวเลขสำคัญ (Summary Stats Cards)
ใช้การ์ด 4-5 ใบแสดงตัวเลขขนาดใหญ่พร้อม Trend (เช่น +5% จากสัปดาห์ก่อน):
1. **Active Visitors**: จำนวนคนที่อยู่ในห้องสมุดตอนนี้ (ดึงจาก `library_visits` status: `active`)
2. **Active Loans**: จำนวนเล่มที่ถูกยืมอยู่ในขณะนี้ (เน้นสี Sky)
3. **Overdue Books**: จำนวนเล่มที่เกินกำหนดและต้องติดตาม (เน้นสี Rose/Red)
4. **Available Items**: จำนวนเล่มที่พร้อมให้บริการ (เน้นสี Emerald/Green)
5. **Pending Fines**: ยอดค่าปรับที่ยังเก็บไม่ได้ (เน้นสี Amber/Orange)

### 3.2 ส่วนกลาง: รายการที่ต้องเร่งดำเนินการ (Actionable Tasks)
แบ่งเป็นแถบรายการ (List) ที่บรรณารักษ์ต้องจัดการ:
- **Reservation Ready**: รายชื่อหนังสือที่จองไว้และพร้อมให้สมาชิกมารับ (เรียงตามวันหมดเขต `holdUntil`)
- **New Member Verification**: รายชื่อสมาชิกใหม่ที่สมัครผ่านหน้าเว็บและรอยืนยันตัวตน
- **Damaged Books Alert**: หนังสือที่ถูกส่งคืนในสภาพ `damaged` และรอการซ่อมแซม

### 3.3 ส่วนล่าง: กิจกรรมล่าสุดและทางลัด (Activities & Shortcuts)
- **Recent Activities**: ตารางแสดงรายการ ยืม-คืน และ การเช็คอิน ล่าสุด 5-10 รายการ
- **Quick Action Buttons**: 
    - [ลงทะเบียนหนังสือใหม่] -> `/manage/register_books`
    - [เพิ่มรหัสเล่มลูก] -> `/manage/add_book_items`
    - [นำเข้าสมาชิกแบบกลุ่ม] -> `/manage/users/import`
    - [จัดการพิกัด/นโยบาย/เวลาทำการ] -> `/manage/settings`

## 4. แผนผังความเชื่อมโยงข้อมูล (Data Connectivity)

| ส่วนประกอบ | ฟังก์ชัน Backend ที่เกี่ยวข้อง | ตารางที่เข้าถึง |
| :--- | :--- | :--- |
| **Stats Summary** | `getDashboardSummary_()` | `loans`, `book_items`, `users`, `fines`, `library_visits` |
| **Pending List** | `getPendingTasks_()` | `reservations`, `users`, `book_items` |
| **Recent Activity** | `getRecentActivities_()` | `loans`, `library_visits` |

---

## 5. ตรรกะการประมวลผลพิเศษ (Special Logic)
- **Caching Strategy**: ข้อมูลสรุปสถิติควรมีการทำ Cache ที่ฝั่ง Browser เป็นเวลา 5-10 นาที เพื่อลดภาระการเรียก API ของ Google Apps Script
- **Background Refresh**: เมื่อมีการทำรายการยืม-คืนสำเร็จในหน้าอื่น ระบบ Dashboard ควรได้รับการสะกิด (Trigger) ให้โหลดข้อมูลใหม่เมื่อ User กลับมาที่หน้านี้

---

## 6. ความสัมพันธ์กับเอกสารอื่น
- **USERS_MANAGEMENT_DESIGN.md**: สำหรับส่วนการยืนยันสมาชิกใหม่
- **BOOKS_MANAGEMENT_DESIGN.md**: สำหรับส่วนสถานะหนังสือชำรุด
- **LOAN_SYSTEM_DESIGN.md**: สำหรับสถิติการยืม-คืน
- **RESERVATION_SYSTEM_DESIGN.md**: สำหรับรายการหนังสือที่พร้อมรับ
- **LIBRARY_VISIT_SYSTEM_DESIGN.md**: สำหรับสถิติผู้เข้าใช้ห้องสมุด
