# 🚀 รายงานการแก้ไขปัญหา GPS หาสัญญาณช้า / ไม่เสถียร (Loan Self Service)

**อ้างอิงไฟล์:** `public/app/views/member/loan_self.view.js`

## 🕵️ สาเหตุของปัญหา (Root Cause Analysis)
จากการวิเคราะห์โค้ด พบว่ามี 2 จุดหลักที่ทำให้ผู้ใช้ติดอยู่ที่หน้า "กำลังรอรับพิกัด... accuracy 0m":

1. **`maximumAge: 0` (บังคับหาพิกัดใหม่ 100%):** 
   การตั้งค่านี้ใน `watchPosition` บังคับให้มือถือต้องเปิดชิป GPS และเชื่อมต่อดาวเทียมใหม่ทุกครั้ง **ห้ามใช้พิกัดเดิมที่มีอยู่แล้วเด็ดขาด** ซึ่งเมื่ออยู่ในอาคาร (Indoor) จะใช้เวลานานมาก หรืออาจหาไม่เจอเลยจนกว่าจะหมดเวลา (Timeout)
2. **`enableHighAccuracy: true`:** 
   แม้จะแม่นยำขึ้น แต่แลกมากับความเร็วที่ช้าลงมาก หากชิปประมวลผลไม่ทันจะคืนค่าเป็น 0 หรือ Timeout

## 🔗 ความเชื่อมโยงกับระบบจัดการพิกัด (Manage Settings) ‼️ สำคัญ ‼️
จากหน้าตั้งค่า `http://localhost:5000/manage/settings` แอดมินสามารถกำหนด "รัศมียืม" (`range_borrow`) และ "รัศมีคืน" (`range_return`) ได้

ตรรกะในระบบหลังบ้าน (Apps Script `Module_LoansFines.gs`) ใช้สมการตรวจสอบดังนี้:
> **`ระยะห่าง (Distance) + ความคลาดเคลื่อน GPS (Accuracy)  <=  รัศมีที่อนุญาต (Range)`**

**💡 ข้อควรระวังขั้นวิกฤต:**
การปรับค่า `GEO_STABLE_ACCURACY` ในโค้ดหน้าบ้าน **ต้องสัมพันธ์กับรัศมีที่แอดมินตั้งไว้** 
ตัวอย่างเช่น: หากเราปรับให้แอปยอมรับ Accuracy ที่ 80m แต่แอดมินตั้ง "รัศมียืม" ในระบบไว้แค่ 50m... ผู้ใช้จะไม่มีทางผ่านระบบนี้ได้เลย (เพราะ 0 + 80 <= 50 เป็นเท็จเสมอ) ระบบจะฟ้องว่า "อยู่นอกเขตที่อนุญาต" ทันทีแม้จะยืนอยู่ตรงจุดนั้นเป๊ะๆ ก็ตาม

## 🛠 วิธีแก้ไขโค้ดที่แนะนำ & การประเมินความเสี่ยง (Review & Solution)

### ✅ จุดที่ 1: อนุญาตให้ใช้พิกัดที่เพิ่งหามา (Cached Location) - [ความเสี่ยง: ต่ำ]
**แนะนำให้ทำทันที:** เลื่อนไปที่บรรทัดประมาณ 897 ในฟังก์ชัน `navigator.geolocation.watchPosition` 

**🔴 โค้ดเดิม:**
```javascript
{ enableHighAccuracy: true, timeout: GEO_TIMEOUT_MS, maximumAge: 0 }
```

**🟢 แก้ไขเป็น:**
```javascript
{ enableHighAccuracy: true, timeout: GEO_TIMEOUT_MS, maximumAge: 15000 }
```
> **เหตุผล (Low-risk, High-reward):** การใช้ cached location 15 วินาทีแทบไม่มีผลเสีย ช่วยแก้ปัญหาค้าง 0m แบบ Instant Load

### ⚠️ จุดที่ 2: ปรับเพิ่มความผ่อนปรนของการรับพิกัด (Tolerance) - [ความเสี่ยง: กลาง-สูง]
**คำแนะนำการปรับค่า Accuracy (ต้องสอดคล้องกับหน้า Settings):**

| ค่าตัวแปร (Constant) | ค่าเดิม | ค่าแนะนำใหม่ | ระดับความเสี่ยง | หมายเหตุ |
| :--- | :--- | :--- | :--- | :--- |
| `GEO_FASTPASS_ACCURACY` | 22 | **30** | ต่ำ | ขยับขึ้นเล็กน้อยเพื่อให้ผ่านได้ไวขึ้น |
| `GEO_STABLE_ACCURACY` | 45 | **45 (หรือตาม Range ที่ตั้ง)** | กลาง | **คำเตือน:** ค่านี้ต้อง **น้อยกว่าหรือเท่ากับ** "รัศมียืม" ขั้นต่ำที่ตั้งไว้ในหน้า Settings เสมอ |
| `GEO_MAX_SAMPLE_ACCURACY`| 150 | **150** (คงเดิม) | สูง | **ไม่แนะนำให้ปรับเป็น 200m** เพราะจะกลายเป็นพิกัดขยะ (Garbage data) |

### 💡 ข้อเสนอแนะเชิงปฏิบัติ (Action Plan)
1. **แก้โค้ด Front-end:** เปลี่ยน `maximumAge: 0` เป็น `15000` ทันที
2. **การตั้งค่าในระบบ Manage:** แอดมินควรพิจารณาตั้งค่า "รัศมียืม" (`range_borrow`) ในหน้า `manage/settings` ให้อย่างน้อย **100m - 150m** เพื่อชดเชยการแกว่งของ GPS ภายในอาคาร หากตั้งแคบเกินไป (เช่น 20m) ต่อให้แอปจับพิกัดได้ก็อาจจะถูกหลังบ้านปฏิเสธอยู่ดี
